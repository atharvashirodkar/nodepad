# Contributing to Nodepad

Thanks for your interest in contributing! This file is a short guide with the essentials to help contributors get started quickly.

How to contribute

- Open an issue to discuss larger changes or feature ideas before implementing them.
- For small fixes or improvements, open a pull request with a concise description of the change.

Guidelines

- Keep changes focused and small when possible.
- Follow consistent code style — the project is small, so aim for clear, readable JavaScript.
- Add or update documentation when behavior changes.

Development workflow

1. Fork the repository and create a branch for your change.
2. Make your changes and ensure they run locally (`npm install` then `npm start`).
3. Open a pull request describing the purpose and scope of the change.

Where to get help

- Use the repository Issues to ask questions or request feedback before submitting large PRs.

Thanks — maintainers will review pull requests and provide feedback.
